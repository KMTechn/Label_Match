# Label Match 작업 중심 UI 결정 목업

기준 해상도는 `1440×900`이며 승인된 작업대는 실제 5단계 업무 계약을 사용한다.

## 결정

`option-1-three-column.svg`를 권장·선정안으로 확정한다.

- 좌측: 현재 품목·작업자·최소 설정
- 중앙: 현재 단계, 다음 행동, 5단계 레일, 단일 안내/경고, 스캔 입력, 실제 현재 세트 스캔 목록
- 우측: 이번 세션·기록 조회·누적 탭과 후속 작업, F1~F4 2×2 버튼
- 중앙 하단 목록은 목업용 요약 카드가 아니라 실제 수락된 `current_set_info["raw"]` 스캔 5행을 단계·값·상태와 함께 계속 표시한다. `parsed` 값은 업무 판정과 품목 식별 계약에만 사용한다.
- 목록의 선택 행 아래에는 `단계`와 `상태`를 문구로 명시하고, 잘리지 않은 `raw` 전체값을 읽기 전용·자동 줄바꿈 상세로 표시한다. 마우스 선택과 키보드 행 이동은 같은 상세를 갱신하며, 리렌더·완료 스냅샷·복구 뒤에도 유효한 선택을 보존한다.
- F4 전체 재스캔은 QA 5단계와 합치지 않고 `m/N` 별도 탭과 배지로 표현한다.
- 오류 문구는 중앙의 단일 경고 영역에만 표시하고, 실패 입력은 정상 목록에 추가하지 않는다.

## 상태 표현 계약

- 실제 단계: `현품표 → 제품 1 → 제품 2 → 제품 3 → 최종 라벨`
- sealed 입력: `이적 QR · 전체 멤버십 서버 상속` 배지로 표시하고 F4를 비활성화한다.
- F4 진행: 중앙 목록의 `F4 전체 재스캔 m/N` 탭을 활성화하되 QA 진행은 `1/5`로 유지한다.
- F3 부분 완료: `소량 완료`라고 명시하며 정상 5/5 완료와 문구를 구분한다.
- 복구: `복구됨` 배지를 유지하고 복원된 실제 스캔 행을 그대로 표시한다.
- 오류: 단일 경고 영역에서만 상세를 표시하고 마지막 정상 스캔 행을 보존한다.
- 완료: 비즈니스 상태가 초기화된 뒤에도 UI 전용 직전 완료 스냅샷을 다음 유효 현품표 입력 전까지 표시할 수 있다.

## 반응형 적용 기준

선정안의 목표 열 폭은 다음과 같다.

| 해상도 | 좌측 | 중앙 | 우측 | 외곽/열 간격 |
|---|---:|---:|---:|---:|
| `1366×768` | 232px | 668px | 422px | 12px / 10px |
| `1440×900` | 248px | 704px | 436px | 14px / 12px |
| `1920×1080` | 300px | 980px | 572px | 18px / 16px |

세 해상도에서는 3열과 F1~F4 2×2 배치를 유지한다. 현재 세트 목록은 가로 스크롤 없이 단계·역할·상태·표시값을 보여주고, 긴 원문은 선택 행 상세에서 제공한다.
